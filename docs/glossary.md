# ユビキタス言語定義（glossary.md）

## 前提

- 本ドキュメントは、`product-requirements.md` / `functional-design.md` / `development-guidelines.md` に登場する用語を整理し、**ビジネス側（日本語）とコード側（英語）の対応を統一する**ために作成する。
- `functional-design.md` のER図に登場するテーブル名・カラム名は仮称であり、実際の基幹システムDBスキーマ調査後に確定する（`functional-design.md` の [要確認] を参照）。本ドキュメントの英語対応も、DB調査後に見直す前提とする。

## ドメイン用語の定義

業務データの中心となるエンティティ（`functional-design.md` のER図に対応）。

| 日本語 | 英語（コード上の型名） | 定義 |
|---|---|---|
| 担当者 | `Rep`（Sales Representative） | 売上を計上する営業担当者。アプリユーザーと1対1、または1対多で紐づく |
| 得意先 | `Customer` | 商品・サービスを販売する取引先 |
| 仕入先 | `Supplier` | 商品・サービスを仕入れる取引先 |
| 売上実績 | `Sale` | 得意先・担当者に紐づく売上の実績データ |
| 予算 | `Budget` | 担当者ごとに割り当てられる、対象年月の売上目標金額 |
| 請求 | `Invoice` | 得意先に対して発行する請求書 |
| 入金 | `Payment` | 得意先からの入金実績 |
| 売掛金 | `Receivable` | 得意先に対して未回収の債権残高 |
| 返品 | `Return` | 得意先からの返品（件数・金額） |
| 値引 | `Discount` | 得意先に対する値引（金額） |
| 見積 | `Quote` | 得意先に対する見積（第2目標で価格入力・更新の対象） |
| 見積明細 | `QuoteLine` | 見積に含まれる品目単位の明細（品目・数量・単価・金額） |
| アプリユーザー | `AppUser` | 本アプリにログインするためのアカウント（担当者コードと紐づく） |

## ビジネス用語の定義

| 用語 | 定義 |
|---|---|
| 第1目標 | 本プロダクトの現行スコープ。ログイン認証を除く全メニューが参照（閲覧）のみの機能 |
| 第2目標 | 第1目標の後に追加する更新系機能（見積価格設定）。基幹システムへの書き込みを伴う |
| 売掛金残高 | ある得意先に対して、請求済みかつ未回収の金額の合計 |
| 予算進捗（率） | ある担当者の対象年月の予算金額に対する、実際の売上実績の達成割合 |
| 担当者別売上対比 | 月初から前日までの売上実績を、担当者間または前年同期等と比較した数値 |
| 入金消込 | 入金実績を対応する請求に突き合わせる処理（本アプリでは参照のみ、消込処理自体は基幹システム側で行う想定）
  > **[要確認]** 消込処理の主体・範囲は基幹システム側の仕様に依存するため、DB調査時に確認する

## UI/UX用語の定義

| 用語 | 定義 |
|---|---|
| メニュー | ログイン後に表示される、13機能（`product-requirements.md` の機能一覧）へのナビゲーション項目 |
| グローバルナビゲーション | 全画面共通で表示される、メニュー一覧・ログインユーザー名・ログアウトを含むヘッダー領域 |
| ハンバーガーメニュー | スマートフォン表示時に、グローバルナビゲーションを折りたたんで表示するUIパターン |
| 一覧 | 複数件のデータを表形式で表示する画面・コンポーネント（例: 顧客情報一覧） |
| 詳細 | 一覧から特定の1件を選択した際に表示される、個別データの画面 |
| 絞り込み（フィルタ） | 期間指定・担当者選択・得意先検索など、一覧の表示件数を条件で絞り込む操作 |
| 推移グラフ | 月次の値の変化を折れ線・棒グラフで表示するコンポーネント（担当者別売上推移、県別売上推移等） |
| 進捗表示（ゲージ） | 予算に対する実績の達成率を、円形のゲージ（`RadialBarChart`）で表示するコンポーネント |

## 英語・日本語対応表

コード上のプロパティ名・関数名で使用する対応。`development-guidelines.md` の命名規則（camelCase）に従う。

| 日本語 | 英語（コード上のプロパティ名） | DB物理カラム名（判明分） |
|---|---|---|
| 得意先コード | `customerCode` | `得意先CD`（`ET0020得意先`） |
| 得意先名 | `customerName` | `得意先名`（`ET0020得意先`） |
| 都道府県 | `prefecture` | `県CD`（`ET0020得意先`）→ `県名`（`ET0001県`） |
| 仕入先コード | `supplierCode` | `仕入先CD`（`SV0020仕入先`、実体は別DBの`T430仕入先`） |
| 仕入先名 | `supplierName` | `仕入先名`（`SV0020仕入先`） |
| 担当者コード | `repCode` | `担当者CD`（`ET0010担当者`, smallint） |
| 担当者名 | `repName` | `担当者名`（`ET0010担当者`） |
| 売上日 | `saleDate` | `売上日`（`ET0120請求明細`） |
| 売上年月 | `salesMonth` | `売上年月`（`ET0100担当者別売上`等） |
| 金額 | `amount` | `販売金額` / `売上金額`等（テーブルにより異なる） |
| 予算金額 | `budgetAmount` | `10月`〜`9月`の各列（`ET0170営業担当予算`、ピボット形式） |
| 請求書番号 | `invoiceNo` | `請求番号`（`ET0110請求`） |
| 請求日 | `invoiceDate` | `請求日`（`ET0110請求`） |
| 請求金額 | `invoiceAmount` | `今回請求残高`（`ET0110請求`） |
| 入金日 | `paymentDate` | `入金日`（`ET0150入金`） |
| 入金額 | `paymentAmount` | `入金額`（`ET0150入金` / `ET0160入金確認`） |
| 売掛金残高 | `receivableBalance` | `今回売掛残高`（`ET0130売掛`） |
| 返品 | `return` | `ET0120請求明細`の`区分='4'`の行（推測、要確認） |
| 値引金額 | `discountAmount` | `ET0120請求明細`の`区分`IN(`'6'`,`'7'`)の行の`販売金額`（推測、要確認） |
| 見積番号 | `quoteNo` | 該当テーブルなし（第2目標で新規設計） |
| 品目 | `item` | `商品名`（`ET0120請求明細`） |
| 数量 | `quantity` | `受注数量`（`ET0120請求明細`） |
| 単価 | `unitPrice` | `販売単価`（`ET0120請求明細`） |
| ユーザーID | `userId` | `ユーザーID`（`app.AppUser`、本アプリ専用テーブル） |
| パスワードハッシュ | `passwordHash` | `パスワードハッシュ`（`app.AppUser`） |

DBスキーマ調査（2026-07-10実施）により、上記は実際の基幹システム（`SaleDB`）のテーブル・カラム名で確認済み。詳細な物理テーブル対応表・返品/値引コードの解釈（要確認）は`functional-design.md`の「データモデル定義（実テーブル対応表）」を参照。

## コード上の命名規則

コード上の命名ルール自体（camelCase / PascalCase の使い分け等）は `development-guidelines.md` の「命名規則」に定義済みのため、本ドキュメントでは重複を避け参照のみとする。本ドキュメントは「どの日本語の業務用語が、コード上のどの英語表現に対応するか」の**対応表**として機能させる。
